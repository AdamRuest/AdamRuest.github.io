Memory Game

This project was done a a mid-term project for my C++ class back in Februaury of 2022. The goal was to make a small mini-game that would consist of showing up a sequence of numbers for a delimited amount of time. The user would then hae to re-write the sequence perfectlty otherwise he'll lose a life. The user had 3 life before losing and then he'd get a new sequence for hime to try to remeber. The users were kept in a text file, so soemone could long on their account and resart where they left off last time, their level determining the amount of number the sequence would have. Doing this project was the first time I did a big project in C++ and was the first time one of my project used a lot of library, 11 to be exact and most of them where not even for logic, just formatting and making the thing look good.

This project led me to learn how to use the windows command prompt at first to execute a project outside of Visual Studio. This being something I never did before, I had to learn how to do it, but... well... the Internet is a thing. This project was also the first time I worked with header files, these being very useful in C++ command prompt programming, allowing you to import functions made by other peoples and not being confinned to the boudaries of the basic libraries like "iostream" or "string". The C++ community is a very large one, and if you need to do somethign that you don't know how to do, you can probably find a header file somewhere on the internet that could help you. This feature is overall very useful for any programmer.

In terms of problem I encountered along the way, I did not face any except for one. One of the required things we had to do was to add sound to the game, indicating if the player won or lost. We were supposed to do it with the "Windows.h" library with the "beep" command. This was quite easy to figure out, but I wanted to play some custom sound effects like a ding from an MP4 file. That was the though part, I spent close to 3 days trying to figure out how to do it, before finally finding out that you needed to import the needed library in the project setting themself. This was somthing i never encountered befor and was quite relieved when I got it working. From there on out, it was smooth sailing until the end.
